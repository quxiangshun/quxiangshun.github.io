NetBox 核心功能 ​

编辑日期：2026-02-13

NetBox 提供了丰富的功能，用于管理网络基础设施。本文档将详细介绍 NetBox 的核心功能及其使用方法。

1. IP 地址管理 (IPAM) ​

IPAM 是 NetBox 的核心功能之一，用于管理 IPv4 和 IPv6 地址空间。

1.1 IP 地址管理 ​

  • IP 地址分配：分配和跟踪 IPv4/IPv6 地址
  • 地址状态管理：标记地址为已分配、已预留、已使用等状态
  • 地址历史：跟踪地址的分配和释放历史
  • 地址搜索：快速搜索和过滤 IP 地址

1.2 前缀管理 ​

  • CIDR 前缀管理：管理网络前缀和子网
  • 前缀层次结构：创建和管理前缀的层次结构
  • 前缀状态：标记前缀为可用、已分配、已保留等状态
  • 前缀利用率：查看前缀的利用率和可用地址数量

1.3 VRF 管理 ​

  • VRF 创建：创建和管理虚拟路由转发实例
  • 路由目标：管理 VRF 的路由目标
  • VRF 关联：将 IP 前缀和地址关联到特定 VRF

1.4 VLAN 管理 ​

  • VLAN 创建：创建和管理 VLAN
  • VLAN 组：将相关 VLAN 组织到组中
  • VLAN 关联：将 VLAN 关联到站点和前缀

2. 数据中心基础设施管理 (DCIM) ​

DCIM 功能用于管理物理网络基础设施，包括设备、机架、线缆等。

2.1 站点管理 ​

  • 站点创建：创建和管理物理站点
  • 区域管理：将站点组织到区域中
  • 站点组：将相关站点组织到组中
  • 站点信息：存储站点的详细信息，如地址、联系人等

2.2 机架管理 ​

  • 机架创建：创建和管理物理机架
  • 机架角色：为机架分配角色（如网络、服务器、存储）
  • 机架位置：在站点中精确定位机架
  • 机架单位：管理机架中的设备位置
  • 机架 Elevation：可视化查看机架中的设备布局

2.3 设备管理 ​

  • 设备创建：创建和管理网络设备
  • 设备类型：定义设备的物理属性和接口模板
  • 设备角色：为设备分配角色（如核心交换机、接入交换机）
  • 设备位置：在机架中定位设备
  • 设备状态：标记设备为在线、离线、计划中、故障等状态

2.4 接口管理 ​

  • 接口创建：为设备创建和管理网络接口
  • 接口类型：支持各种接口类型（如以太网、光纤、Console）
  • 接口状态：标记接口为已连接、未连接、计划中
  • 接口配置：管理接口的 IP 地址和 VLAN 配置

2.5 线缆管理 ​

  • 线缆创建：创建和管理设备间的物理连接
  • 线缆类型：支持各种线缆类型（如 Cat6、光纤）
  • 连接跟踪：跟踪设备间的连接关系
  • 线缆长度：记录线缆的物理长度
  • 连接可视化：通过拓扑图查看设备间的连接

2.6 电力管理 ​

  • 电力面板：管理站点的电力面板
  • 电力馈线：管理从电力面板到机架的电力馈线
  • 电力插座：管理机架中的电力插座
  • 设备电源端口：管理设备的电源输入端口
  • 电力连接：跟踪设备的电力连接

3. 虚拟ization 管理 ​

虚拟化功能用于管理虚拟环境，包括虚拟机和集群。

3.1 集群管理 ​

  • 集群创建：创建和管理虚拟化集群
  • 集群类型：定义集群的类型（如 VMware vSphere、KVM）
  • 集群站点：将集群关联到物理站点

3.2 虚拟机管理 ​

  • 虚拟机创建：创建和管理虚拟机
  • 虚拟机状态：标记虚拟机为活动、离线、计划中、故障等状态
  • 虚拟机接口：管理虚拟机的网络接口
  • 虚拟机到设备关联：将虚拟机关联到物理主机设备

4. 电路管理 ​

电路管理功能用于管理与服务提供商的外部连接。

4.1 提供商管理 ​

  • 提供商创建：创建和管理服务提供商
  • 提供商账户：管理与提供商的账户信息

4.2 电路管理 ​

  • 电路创建：创建和管理外部电路
  • 电路类型：定义电路的类型（如互联网、专线）
  • 电路状态：标记电路为活动、计划中、停用等状态
  • 电路termination：管理电路的两端连接点

5. API 集成 ​

NetBox 提供了强大的 API，用于与其他系统集成。

5.1 REST API ​

  • 完整 API：所有 NetBox 功能都通过 REST API 暴露
  • API 版本控制：支持 API 版本控制
  • API 认证：支持 Token 认证和会话认证
  • API 文档：提供交互式 API 文档（基于 Swagger）

5.2 GraphQL API ​

  • GraphQL 支持：提供 GraphQL API 端点
  • 灵活查询：允许客户端指定需要的数据
  • 嵌套查询：支持复杂的嵌套数据查询

5.3 Webhooks ​

  • Webhook 创建：创建和管理 webhook
  • 事件触发：基于 NetBox 事件触发 webhook
  • 自定义 payload：自定义 webhook 的 payload 格式
  • HTTP 方法：支持 POST、PUT、PATCH 等 HTTP 方法

6. 自定义功能 ​

NetBox 提供了多种自定义功能，以满足特定需求。

6.1 自定义字段 ​

  • 自定义字段创建：为任何对象类型创建自定义字段
  • 字段类型：支持文本、整数、布尔值、选择等字段类型
  • 字段验证：为自定义字段添加验证规则

6.2 标签 ​

  • 标签创建：创建和管理标签
  • 标签关联：将标签关联到任何对象
  • 标签过滤：使用标签过滤对象

6.3 报告 ​

  • 报告创建：创建自定义报告
  • 报告执行：手动或自动执行报告
  • 报告结果：查看报告的执行结果和发现的问题

6.4 脚本 ​

  • 脚本创建：创建自定义脚本
  • 脚本执行：手动执行脚本
  • 脚本参数：为脚本定义参数

7. 搜索和过滤 ​

NetBox 提供了强大的搜索和过滤功能，便于查找和管理对象。

7.1 全局搜索 ​

  • 全文搜索：在整个 NetBox 实例中搜索对象
  • 搜索结果：按对象类型分组显示搜索结果
  • 搜索语法：支持高级搜索语法

7.2 列表视图过滤 ​

  • 高级过滤：在列表视图中使用高级过滤条件
  • 过滤保存：保存常用的过滤条件
  • 批量操作：对过滤结果执行批量操作

8. 变更日志和审计 ​

NetBox 自动记录所有对象的变更，提供完整的审计跟踪。

8.1 变更日志 ​

  • 变更记录：自动记录所有对象的创建、更新和删除
  • 变更详情：查看变更的详细内容和差异
  • 变更作者：记录执行变更的用户
  • 变更时间：记录变更的时间戳

8.2 审计日志 ​

  • 用户活动：记录用户的登录和操作活动
  • API 访问：记录 API 访问和操作
  • 认证事件：记录认证成功和失败事件

9. 通知系统 ​

NetBox 提供了灵活的通知系统，用于提醒用户重要事件。

9.1 通知创建 ​

  • 通知类型：创建和管理通知类型
  • 通知触发：基于事件触发通知
  • 通知目标：指定通知的接收者

9.2 通知方法 ​

  • Web 界面：在 NetBox Web 界面中显示通知
  • Email：通过电子邮件发送通知
  • Webhook：通过 webhook 发送通知到外部系统

10. 插件系统 ​

NetBox 支持插件，用于扩展核心功能。

10.1 插件安装 ​

  • 插件安装：安装和管理第三方插件
  • 插件配置：配置插件的设置
  • 插件启用/禁用：启用或禁用插件

10.2 插件开发 ​

  • 插件框架：基于 Django 插件架构
  • 插件 API：使用 NetBox 插件 API 扩展功能
  • 插件文档：提供插件开发文档
